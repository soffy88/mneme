## 1. 产品概述

### 1.1 产品定位

**高考作战指挥系统**不是一个试卷分析工具，而是一个**面向高考生和家长的AI智能备考作战平台**。

核心价值主张：

> 别人告诉学生哪里错了。我们告诉学生——剩下的时间，该怎么活。

### 1.2 产品背景

现有备考工具的根本缺陷：

- **只有诊断，没有决策**：告诉学生"解析几何很弱"，但不告诉他值不值得花时间补、还来不来得及
- **只有数据，没有陪伴**：雷达图和错题列表是冷冰冰的数字，无法回应高考生的焦虑和崩溃
- **只有学生端，没有家长端**：家长的焦虑来自"不透明"，现有产品无法给家长真正有用的信息
- **只有计划，没有执行支持**：给了作战清单，但学生拖延症发作时无人推一把
- **只有列表，没有遗忘对抗**：错题本是静态的，不知道该什么时候回来复习

### 1.3 融合设计理念

本产品融合三个核心系统：

- **试卷分析系统**：OCR识别 + AI批改 + 知识点画像（数据入口）
- **苏格拉底教学引擎**：不给答案、追问引导、认知修复（深度学习）
- **执行力支持系统**：番茄钟、遗忘曲线、Streak、智能推送（持续坚持）

三者形成完整闭环：试卷是数据入口，苏格拉底修复认知，执行力系统保障每天真正动起来。

## 2. 用户研究

### 2.1 目标用户

**主要用户：高考生**
- 年龄：16～18岁，高三在读
- 核心焦虑：时间不够用、不知道该重点复习什么、担心考不上目标院校
- 行为特征：碎片化学习、间歇性自律、对结果极度敏感、拖延症普遍存在

**次要用户：高考生家长**
- 年龄：40～50岁
- 核心焦虑：不知道孩子真实水平、不知道自己能做什么、害怕帮倒忙
- 行为特征：高焦虑、信息饥渴、愿意为孩子付费、主要通过微信获取信息
- 家庭结构：二孩家庭比例上升，需支持多孩子管理

### 2.2 用户痛点深度分析

#### 高考生的真实心声

| 痛点 | 具体表现 | 现有解决方案的不足 |
|------|---------|-----------------|
| 不知道该学什么 | 什么都想学，什么都没学好 | 工具只列清单，不做决策 |
| 不知道自己排名 | 520分能上什么学校？ | 只有自我对比，无横向定位 |
| 学了还是不会 | 看了答案，下道题还是错 | 只给答案，不修复认知 |
| 越学越焦虑 | 刷题刷到崩溃 | 无情绪感知机制 |
| 拖延症发作 | 计划做了，执行不了 | 无主动触达和激励机制 |
| 错题学了又忘 | 上周错的题，这周还是错 | 错题本是静态列表，无遗忘曲线管理 |
| 太累不想打字 | 想问问题，但懒得输入 | 纯文字交互，摩擦力大 |
| 首页信息太多 | 打开APP不知道先干啥 | 仪表盘复杂，疲惫时认知负担重 |

#### 家长的真实心声

| 痛点 | 具体表现 | 现有解决方案的不足 |
|------|---------|-----------------|
| 信息黑洞 | 不知道孩子今天学了什么 | 家长端要么没有，要么只有分数 |
| 无法判断形势 | 现在这个水平能考上吗？ | 没有预估分和录取概率 |
| 不知道如何帮 | 该催还是不催？ | 没有家长行动指南 |
| 担心孩子状态 | 关门学习，情绪看不出来 | 无情绪预警机制 |
| 不用APP也想知道 | 每天不会专门打开一个新APP | 推送依赖APP，微信触达率更高 |
| 两个孩子要管 | 老大高三，老二高一，来回切换麻烦 | 单账号单孩子设计 |
| 现在就想知道能上哪 | 高三上学期就开始焦虑志愿 | 志愿参考出现太晚 |

### 2.3 用户场景

**场景一：疲惫状态下的极简启动**
> 小明晚上10点刷完题，很累，打开APP。首屏只有一句话："今天只做这一件事：完成椭圆专项追问（预计20分钟）"。他点进去，完成了，系统显示"今天完成了，明天继续"。他关掉APP去睡觉。没有多余的仪表盘让他焦虑。

**场景二：单题快速录入 + 30秒首次追问**
> 做题途中，小明遇到一道不会的解析几何题。拍照，选科目，确认。30秒内系统返回第一个苏格拉底追问："这道题告诉你椭圆的焦距是多少？先找出来。"他马上开始思考，不需要等待。

**场景三：遗忘曲线自动回顾**
> 小明三天前学了"椭圆焦点三角形"，今天系统推送："你有1道题到了回顾时间，现在做一下，5分钟就够。"他完成后，系统根据表现决定下次回顾时间——答对了推迟7天，答错了明天再来。

**场景四：家长微信日报**
> 小明妈妈晚上9点收到微信消息："【今日学情】小明今天学了2h35m，完成椭圆专项，情绪稳定。建议：今晚不要问成绩，他正处于正向状态。"她放心了，没有去催孩子。

**场景五：高三上学期志愿预判**
> 小明爸爸打开家长端，看到："基于目前3次模考，小明预估分538分（广东省）。当前水平对应：冲刺—中山大学，稳妥—华南理工大学，保底—暨南大学。距高考还有180天，仍有较大提升空间。"他心里有了底，不再天天催孩子。

## 3. 产品目标与成功指标

### 3.1 产品目标

**短期目标（MVP，0-3个月）**：验证核心价值，获取第一批付费用户

**中期目标（3-12个月）**：建立用户学习数据积累，形成差异化壁垒

**长期目标（1年+）**：成为高考备考领域的标准化AI作战工具

### 3.2 成功指标（OKR）

#### Objective 1：让学生真正用起来

| 关键结果 | 目标值 | 衡量方式 |
|---------|-------|---------|
| 次日留存率 | ≥ 40% | 用户次日登录率 |
| 周活跃率 | ≥ 60% | 每周至少上传1次试卷或完成1次对话 |
| 单次苏格拉底对话时长 | ≥ 8分钟 | 对话轮次统计 |
| 用户自发分享率 | ≥ 15% | 分享报告到朋友圈/群 |
| Streak 7天达成率 | ≥ 30% | 连续7天完成今日目标的用户比例 |

#### Objective 2：让家长愿意付费

| 关键结果 | 目标值 | 衡量方式 |
|---------|-------|---------|
| 家长端DAU/学生端DAU | ≥ 50% | 家长活跃比例 |
| 付费转化率（家长） | ≥ 25% | 试用转付费比例 |
| 家长NPS | ≥ 50 | 月度调研 |
| 微信日报点击率 | ≥ 40% | 日报中链接点击比例 |

#### Objective 3：AI效果可量化

| 关键结果 | 目标值 | 衡量方式 |
|---------|-------|---------|
| 错题再遇正确率 | ≥ 65% | 相同知识点题目二次测试 |
| 遗忘曲线回顾通过率 | ≥ 70% | 按时回顾并答对的比例 |
| 预估分准确率 | 误差 ≤ 15分 | 与实际模考分对比 |
| OCR识别准确率 | ≥ 95% | 人工抽查验证 |

## 4. 用户旅程地图

### 4.1 学生完整旅程

```
[发现] → [注册] → [首次使用] → [日常使用] → [冲刺阶段] → [考后]

发现
└── 朋友分享学情报告 / 搜索"高考备考工具"

注册
├── 填写：省份、目标院校、目标专业、高考日期
├── 上传最近一次模考成绩（各科分数）
└── 系统立即生成：首份战局评估 + 今日唯一目标

首次使用（关键体验，必须在5分钟内打动用户）
├── 首屏看到：今日唯一目标（一句话 + 预计时长）
├── 上传一张试卷 → 2分钟内看到战局报告
└── 点击一道错题 → 30秒内收到第一个苏格拉底追问

日常使用（习惯养成）
├── 每天完成今日唯一目标 → 积累Streak天数
├── 番茄钟专注模式 → 完成任务解锁成就感
├── 遗忘曲线推送 → 定期回顾错题
├── 随时拍照录入单道错题（碎片化触发）
└── 完成苏格拉底对话 → 弱点画像更新

冲刺阶段（高考前30天）
├── 系统自动切换模式：深度理解 → 模板记忆
├── 每日唯一目标更精简，认知负担最小化
├── 停止追加新弱点，专注已标记高优先级知识点
└── 每周战局复盘：还差多少、还来得及吗

考后
└── 生成完整备考报告（可下载分享）
```

### 4.2 家长旅程

```
注册
├── 绑定孩子账号（输入孩子的6位邀请码）
└── 支持绑定多个孩子，首页可切换

日常使用
├── 每晚微信推送：一句话日报（时长、任务、情绪、建议）
├── 每周推送：战局周报 + 本周家长行动指南
└── 异常推送：5类风险预警（情绪/成绩/任务/时间/睡眠）

关键节点
├── 注册后即可查看：院校层次初步判断（简版志愿参考）
├── 模考后：看到孩子预估分变化趋势
└── 高三下学期：精细化志愿梯度建议
```

### M0：今日唯一目标（首屏核心 🆕）

> **这是v1.1最重要的新增模块。** 学生打开APP的第一眼决定他会不会继续用。

#### M0.1 设计原则

- 首屏**永远只展示一件事**，不展示仪表盘、不展示数据堆砌
- 学生无需思考"我今天该干什么"——系统已经替他决定好了
- 疲惫状态下也能无摩擦启动

#### M0.2 首屏UI结构

```
┌─────────────────────────────────┐
│  距高考 63 天                    │
│                                 │
│  ┌─────────────────────────────┐ │
│  │                             │ │
│  │   今天只做这一件事           │ │
│  │                             │ │
│  │   完成椭圆焦点三角形追问     │ │
│  │   预计 20 分钟               │ │
│  │                             │ │
│  │   [  开始  ]                │ │
│  │                             │ │
│  └─────────────────────────────┘ │
│                                 │
│  已坚持 12 天  🔥                │
│                                 │
│  ──────────────────────────     │
│  拍一道题   查看战局   错题回顾   │
└─────────────────────────────────┘
```

#### M0.3 今日目标生成逻辑

```
优先级排序：
  1. 今日有遗忘曲线到期的错题 → "回顾[X]道到期错题（5分钟）"
  2. 当周必打知识点有未完成对话 → "完成[知识点]苏格拉底追问（20分钟）"
  3. 已超过3天未上传试卷 → "拍一道今天的错题（10分钟）"
  4. 默认 → 当周ROI最高的知识点专项练习

约束：
  - 预计时长不超过30分钟（降低启动门槛）
  - 高考前30天：预计时长不超过15分钟
  - 晚上11点后：目标自动降级为"今天可以了，明天继续"
```

#### M0.4 完成反馈

完成今日目标后：
- 显示简洁的完成动画（不要过度游戏化）
- 更新Streak天数
- 出现"今天完成了"字样 + 明日目标预告（一句话）
- 不强制继续，给予清晰的"可以关掉APP了"信号

### M2：试卷录入系统

#### M2.1 试卷上传（主流程）

**支持格式**：JPG/PNG/PDF，单次最多20页

**处理流程**：
```
上传图片
  → 图像预处理（去噪、矫正、增强对比度）
  → OCR识别（题目结构化：题号、题干、选项、学生答案、标准答案）
  → AI批改（逐题对比，标注正误）
  → Profiler Agent 深度分析（错误类型归因、认知断裂点定位）
  → 更新学生画像
  → 初始化遗忘曲线计划（为每道错题设定首次回顾时间）
  → 生成本次考试报告
```

**错误类型分类**（Profiler Agent输出）：
- **概念性错误**：根本没理解该知识点
- **迁移性错误**：理解了但不会在新场景中应用
- **粗心性错误**：会做但计算/书写失误
- **思维断裂**：推导到某步骤停滞，后续逻辑缺失

**处理时间要求**：
- 单张图片：≤ 60秒
- 完整试卷（20页）：≤ 3分钟
- 超时提示：进度条 + "处理中，可先看其他内容"

#### M2.2 单题快速录入（快捷流程）

**使用场景**：学生做题过程中遇到不会的题，当下立即录入

**交互设计**：
1. 首页底部常驻按钮"拍一道题"
2. 拍照 → 识别题目 → 选择科目 → 确认
3. **30秒内**必须返回第一个苏格拉底追问（硬性SLA，见第9章）
4. 对话结束后自动归入对应科目错题库
5. 同步初始化该题的遗忘曲线计划

#### M2.3 手动录入成绩

**字段**：
- 考试名称（如：2025年3月第一次全市模考）
- 考试日期
- 各科分数 / 总分
- 年级排名（可选）

### M3：战局评估引擎

> 这是产品留存的核心钩子。用户第一次看到战局报告，必须感受到"这个系统真的懂我"。

#### M3.1 当前预估分计算

```
预估分 = Σ(各科预估分)

各科预估分 = 各科满分 × 综合得分率

综合得分率 =
  历次模考得分率（权重40%）× 时间衰减系数
  + 知识点掌握度加权平均（权重40%）
  + 近期错题难度系数修正（权重20%）
```

**显示形式**：
- 预估分：`538分`（区间显示：`525～550分`）
- 各科分解：语文`105` / 数学`112` / 英语`118` / 综合`203`

#### M3.2 战局判断

| 状态 | 判断标准 | 展示颜色 | 展示文案示例 |
|------|---------|---------|------------|
| 绿色 | 预估分 ≥ 目标分 | 绿色 | "按当前进度，你大概率能达到目标，继续保持" |
| 黄色 | 目标分 - 预估分 ≤ 30分 | 黄色 | "还差28分，调整策略可以追上，重点看数学" |
| 红色 | 目标分 - 预估分 > 30分 | 红色 | "差距较大，必须做取舍——不是所有弱点都来得及补" |

#### M3.3 时间分析

```
剩余天数：63天
可用学习时长（估算）：63天 × 4小时/天 = 252小时
已识别需补知识点总时长：约340小时
结论：⚠️ 时间不够，必须取舍
```

### M4：作战决策系统

#### M4.1 知识点ROI计算

```
ROI = (预期得分提升 × 高考权重系数) ÷ 补完所需天数

预期得分提升 = (1 - 当前掌握度) × 该知识点平均分值 × 0.8（保守系数）
高考权重系数 = 该知识点在高考中的历年平均分值占比
补完所需天数 = 基于当前掌握度 + 错误类型估算
```

#### M4.2 知识点三色分级

| 等级 | 判定标准 | 操作建议 |
|------|---------|---------|
| 🟢 必打 | ROI > 6 且剩余时间充足 | 本周优先投入，深度学习 |
| 🟡 选打 | ROI 3～6 或时间紧张 | 根据当周进度决定是否投入 |
| 🔴 放弃 | ROI < 3 或补完时间 > 剩余天数50% | 果断放弃，不要有心理负担 |

**放弃建议措辞原则**（避免打击信心）：
> "数列压轴题：预计需要20天才能稳定掌握，高考只有1道，分值6分。建议本阶段跳过，把这20天用在更高回报的地方。这不是放弃，是聪明的选择。"

#### M4.3 每周作战清单

每周一自动生成，内容精简，不超过5条：

```
📋 本周作战清单（第8周，距高考63天）

🎯 本周必做
  1. 解析几何：椭圆标准方程 + 焦点三角形（3道专项练习）
  2. 英语：完形填空逻辑词汇（每日10题，共50题）

⚡ 如果时间允许
  3. 物理：电磁感应计算（重点做选择题，放弃大题）

🚫 本周不做
  数列压轴题、化学有机推断（性价比过低）

💡 本周家长建议
  数学不需要催，英语每日10题需要监督打卡。
```

#### M4.4 长期作战地图

可视化展示未来每周重点攻坚方向，让学生看到"完整的仗该怎么打"。

### M5：苏格拉底对话引擎

#### M5.1 对话模式切换

| 模式 | 触发条件 | 特征 |
|------|---------|------|
| **深度理解模式** | 距高考 > 90天 | 严格苏格拉底式，不给答案，多轮追问，引导学生自主推导 |
| **混合模式** | 30～90天 | 追问为主，遇到学生卡壳超过3轮时给出部分提示 |
| **冲刺模式** | < 30天 | 快速确认知识点理解 → 直接给出解题模板 → 强化记忆锚点 |

#### M5.2 Tutor Agent 核心规则

**铁律（任何模式下不违反）**：
1. 不直接给出最终答案或完整解题步骤
2. 当学生说"我不会"时，从最基础的概念开始追问
3. 只有当学生自己说出正确思路时，才给予肯定和推进
4. 每次追问只问一个问题，不连续发问

**逃生出口**：
- 学生可随时点击"我想直接看答案"退出苏格拉底模式
- 系统记录使用次数但不惩罚，不影响Streak
- 退出后仍正常记录错题，遗忘曲线照常运行

**追问策略（深度理解模式）**：
```
Step 1：还原题目 — "这道题在问什么？用自己的话说说"
Step 2：拆解条件 — "题目给了你哪些已知信息？"
Step 3：找突破口 — "你觉得第一步可以做什么？"
Step 4：暴露断裂 — 学生推导错误时 → "你这一步的依据是什么？"
Step 5：自我纠正 — 引导学生发现自己的逻辑错误
Step 6：总结归纳 — "你刚才用了什么方法解决这道题？"
```

#### M5.3 情绪感知层

**情绪识别触发词**：
- 焦虑类："学不完了"、"好慌"、"来不及了"、"没用"
- 崩溃类："不想学了"、"想放弃"、"我太差了"
- 愤怒类："这什么破题"、"学了有什么用"

**触发后处理逻辑**：
1. 暂停学术追问
2. 切换到情绪支持回应（非学术语气）
3. 情绪词汇出现 ≥ 3次 → 触发家长端情绪预警推送
4. 提供"今天就到这里吧"退出选项，退出不影响Streak

**情绪支持示例**：
> "听起来你今天压力很大。这很正常，高考前每个人都会有这种感觉。不用现在强迫自己继续——你今天已经做了不少了。要不要先休息一下？"

#### M5.4 语音输入（🆕 新增，Phase 2）

**适用场景**：疲惫时不想打字，或手写数学过程更自然

**技术方案**：
- Web端：Web Speech API（浏览器原生，无需额外依赖）
- App端：系统原生语音输入
- 降级处理：识别失败时提示切换文字输入

**交互设计**：
- 对话界面常驻麦克风按钮
- 长按说话，松开发送
- 语音识别结果展示在输入框，可修改后发送
- 数学公式识别：语音"x平方加2x等于0"→ 自动转换为`x²+2x=0`（尽力而为，识别失败提示手输）

#### M5.5 对话结束后处理

对话结束时Profiler Agent自动运行：
1. 分析本次对话暴露的认知断裂点
2. 更新学生画像中的弱点日志
3. 修正该知识点掌握度评分
4. **更新该题的遗忘曲线计划**（成功推导 → 延长回顾间隔；失败 → 缩短间隔）
5. 重新计算ROI，必要时更新作战清单

### M6：学生画像系统

#### M6.1 知识点体系

基于教育部高考考纲，覆盖6科，详见附录A。

每个知识点记录：
- 掌握度（0.0～1.0）
- 错误类型分布
- 历史弱点日志
- ROI评分
- 分级状态（必打/选打/放弃）
- **遗忘曲线状态**（下次回顾时间、当前记忆强度）

#### M6.2 掌握度更新机制

```
新掌握度 = 旧掌握度 × 0.7 + 本次表现评分 × 0.3

本次表现评分：
  苏格拉底对话中自主推导成功 → 0.85
  苏格拉底对话中需要部分提示 → 0.55
  对话中未能自主推导 → 0.20
  遗忘曲线回顾答对（容易） → 0.70
  遗忘曲线回顾答对（难题） → 0.90
  遗忘曲线回顾答错 → 0.15
  试卷中答对 → 0.75
  试卷中答错 → 0.15
```

#### M6.3 遗忘曲线回顾系统（🆕 新增）

> 不是简单的错题列表。系统自动安排每道错题的回顾时间，对抗遗忘。

**回顾间隔算法**（基于SM-2算法简化版）：

```
初次学完后：
  苏格拉底对话成功 → 1天后回顾
  苏格拉底对话部分成功 → 当天回顾
  直接看答案 → 第二天回顾

每次回顾后：
  答对 → 间隔 × 2（1天→2天→4天→8天→...）
  答错 → 重置为1天后再回顾
  
最大间隔：30天（高考前15天内停止新增回顾，专注冲刺）
```

**回顾题型生成**：
- 不直接复现原题（避免机械记忆）
- 生成同类型、同知识点的变式题（由AI出题）
- 若无AI出题权限（免费用户），展示原题并要求学生遮住答案自测

**回顾触发方式**：
- 每日今日目标优先包含到期回顾题
- 若未触发，当天晚上8点推送提醒
- 回顾时长控制：每次不超过5分钟（最多3道题）

**功能**：

- 不做排行榜

**规则**：
- 每天完成今日唯一目标 = 打卡成功
- 连续打卡天数在首屏醒目展示（"已坚持 12 天 🔥"）
- Streak断了不清零历史最高记录，保留成就感
- 每周日晚提醒当周完成情况："本周完成5/7天，明天继续"

- 每月可使用1次"补卡"机会（漏打一天不断链）
- 高考前7天自动暂停Streak计算，全力冲刺不焦虑

**不做的事**：
- 不做积分商城
- 不做好友排行榜（学习不是竞赛）
- 不做徽章墙（过于幼稚）

**触发条件**：学生在晚上11点后仍在使用APP

**处理逻辑**：
1. 弹出提示："现在是[时间]了，今天做得够多了。睡好觉明天状态更好。"
2. 提供"我知道了，关掉"按钮
3. 不强制退出，不重复弹出
4. 若凌晨12点后仍活跃，触发家长熬夜预警推送

- 语音识别失败：提示切换文字输入，不影响主流程

## 12. 风险评估

| 风险 | 概率 | 影响 | 应对策略 |
|------|------|------|---------|
| OCR对手写数学公式识别率低 | 高 | 高 | MVP阶段以Claude Vision为主，支持用户手动纠错 |

| 学生不愿意被追问，流失 | 中 | 高 | 提供"直接看答案"逃生出口，不惩罚不强制 |
| Streak系统引发焦虑（怕断） | 中 | 中 | 设置补卡机制；高考前7天自动暂停；断了不清零历史最高 |
| 微信模板消息审核不通过 | 中 | 中 | 短信降级方案并行部署；模板内容提前准备多套备选 |
| 家长付费意愿不足 | 中 | 高 | 免费提供战局报告（家长分享传播），付费解锁微信日报和行动指南 |
| 真题版权法律风险 | 高 | 高 | Phase 3前不内置真题文本，AI原创题规避风险；合规合作题库再推出 |

```
└── 空间几何（20分，权重13.3%）
```

### B. 核心术语定义

| 术语 | 定义 |
|------|------|
| 战局评估 | 基于当前学习数据对高考结果的综合预判 |
| 预估分 | AI基于错题分布和历次模考计算的当前能力对应分数 |
| ROI | 知识点学习回报率，衡量投入时间与预期得分提升的比率 |
| 三色分级 | 必打（绿）/选打（黄）/放弃（红）的知识点优先级判定 |
| 苏格拉底模式 | 通过追问引导学生自主推导答案的AI对话方式 |
| 弱点画像 | 学生在各知识点上的认知状态结构化记录 |
| 作战清单 | 每周精简的重点任务列表，包含必做、选做和明确放弃项 |
| 认知断裂点 | 学生推导过程中思维停滞或出现逻辑跳跃的具体位置 |
| 今日唯一目标 | 系统每日为学生生成的一件最重要的事，首屏醒目展示 |
| 遗忘曲线回顾 | 基于Ebbinghaus遗忘曲线，系统自动安排错题复习时间 |
| Streak | 连续完成今日目标的天数，体现坚持的力量 |
| 逃生出口 | 苏格拉底对话中"直接看答案"的退出选项，降低摩擦力 |
| 熬夜预警 | 检测到学生晚11点后仍活跃时，向本人和家长发出的提示 |
| 微信日报 | 每晚发给家长的一句话学情总结，通过微信模板消息推送 |
