## 1. 产品概述

### 1.1 产品定位

**高考作战指挥系统**不是一个试卷分析工具，而是一个**面向高考生和家长的AI智能备考作战平台**。

核心价值主张：

> 别人告诉学生哪里错了。我们告诉学生——剩下的时间，该怎么活。

### 1.2 产品背景

现有备考工具的三大根本缺陷：

- **只有诊断，没有决策**：告诉学生"解析几何很弱"，但不告诉他值不值得花时间补、还来不来得及
- **只有数据，没有陪伴**：雷达图和错题列表是冷冰冰的数字，无法回应高考生的焦虑和崩溃
- **只有学生端，没有家长端**：家长的焦虑来自"不透明"，现有产品无法给家长真正有用的信息

### 1.3 融合设计理念

本产品融合两个核心系统：

- **试卷分析系统**：OCR识别 + AI批改 + 知识点画像（数据入口）
- **苏格拉底教学引擎**：不给答案、追问引导、认知修复（深度学习）

两者形成完整闭环：试卷是数据来源，苏格拉底对话是认知修复机制。

## 2. 用户研究

### 2.1 目标用户

**主要用户：高考生**
- 年龄：16～18岁，高三在读
- 核心焦虑：时间不够用、不知道该重点复习什么、担心考不上目标院校
- 行为特征：碎片化学习、间歇性自律、对结果极度敏感

**次要用户：高考生家长**
- 年龄：40～50岁
- 核心焦虑：不知道孩子真实水平、不知道自己能做什么、害怕帮倒忙
- 行为特征：高焦虑、信息饥渴、愿意为孩子付费

### 2.2 用户痛点深度分析

#### 高考生的真实心声

| 痛点 | 具体表现 | 现有解决方案的不足 |
|------|---------|-----------------|
| 不知道该学什么 | 什么都想学，什么都没学好 | 工具只列清单，不做决策 |
| 不知道自己排名 | 520分能上什么学校？ | 只有自我对比，无横向定位 |
| 学了还是不会 | 看了答案，下道题还是错 | 只给答案，不修复认知 |
| 越学越焦虑 | 刷题刷到崩溃 | 无情绪感知机制 |
| 拖延症发作 | 计划做了，执行不了 | 无主动触达和激励机制 |

#### 家长的真实心声

| 痛点 | 具体表现 | 现有解决方案的不足 |
|------|---------|-----------------|
| 信息黑洞 | 不知道孩子今天学了什么 | 家长端要么没有，要么只有分数 |
| 无法判断形势 | 现在这个水平能考上吗？ | 没有预估分和录取概率 |
| 不知道如何帮 | 该催还是不催？ | 没有家长行动指南 |
| 担心孩子状态 | 关门学习，情绪看不出来 | 无情绪预警机制 |

### 2.3 用户场景

**场景一：日常错题录入**
> 小明做完一套数学模拟卷，有3道题不会。他打开APP，拍照上传试卷，2分钟后看到：当前预估分538分，距目标分580分差42分，本周重点攻克"解析几何选择题"。系统自动为这3道错题开启苏格拉底追问，引导他自己推导出正确思路。

**场景二：家长晚间查看**
> 小明妈妈晚上11点睡前打开家长端，看到今天小明学习了2小时40分钟，完成了3道错题的追问练习，情绪状态稳定，本周建议不要催数学，重点关注英语阅读的每日打卡。

**场景三：高考前30天冲刺**
> 系统自动切换到冲刺模式，苏格拉底追问变为"快速模板记忆"模式，每天推送："今天只做这一件事——背完圆锥曲线的3个解题模板"，降低认知负担，提高执行率。

## 3. 产品目标与成功指标

### 3.1 产品目标

**短期目标（MVP，0-3个月）**：验证核心价值，获取第一批付费用户

**中期目标（3-12个月）**：建立用户学习数据积累，形成差异化壁垒

**长期目标（1年+）**：成为高考备考领域的标准化AI作战工具

### 3.2 成功指标（OKR）

#### Objective 1：让学生真正用起来

| 关键结果 | 目标值 | 衡量方式 |
|---------|-------|---------|
| 次日留存率 | ≥ 40% | 用户次日登录率 |
| 周活跃率 | ≥ 60% | 每周至少上传1次试卷或完成1次对话 |
| 单次苏格拉底对话时长 | ≥ 8分钟 | 对话轮次统计 |
| 用户自发分享率 | ≥ 15% | 分享报告到朋友圈/群 |

#### Objective 2：让家长愿意付费

| 关键结果 | 目标值 | 衡量方式 |
|---------|-------|---------|
| 家长端DAU/学生端DAU | ≥ 50% | 家长活跃比例 |
| 付费转化率（家长） | ≥ 25% | 试用转付费比例 |
| 家长NPS | ≥ 50 | 月度调研 |

#### Objective 3：AI效果可量化

| 关键结果 | 目标值 | 衡量方式 |
|---------|-------|---------|
| 错题再遇正确率 | ≥ 65% | 相同知识点题目二次测试 |
| 预估分准确率 | 误差 ≤ 15分 | 与实际模考分对比 |
| OCR识别准确率 | ≥ 95% | 人工抽查验证 |

## 4. 用户旅程地图

### 4.1 学生完整旅程

```
[发现] → [注册] → [首次使用] → [日常使用] → [冲刺阶段] → [考后]

发现
└── 朋友分享学情报告 / 搜索"高考备考工具"

注册
├── 填写：省份、目标院校、目标专业、高考日期
├── 上传最近一次模考成绩（各科分数）
└── 系统立即生成：首份战局评估报告

首次使用（关键体验，必须在5分钟内打动用户）
├── 上传一张试卷
├── 2分钟内看到：当前预估分、目标分差、本周作战清单
└── 点击一道错题，进入苏格拉底对话体验

日常使用（习惯养成）
├── 每次考试后上传试卷（主动触发）
├── 随时拍照录入单道错题（碎片化触发）
├── 每天收到"今日任务"推送（被动触发）
└── 完成苏格拉底对话 → 弱点画像更新

冲刺阶段（高考前30天）
├── 系统自动切换模式：深度理解 → 模板记忆
├── 每日推送极简任务（1-2件事）
├── 停止追加新弱点，专注已标记高优先级知识点
└── 每周战局复盘：还差多少、还来得及吗

考后
└── 生成完整备考报告（可下载分享）
```

### 4.2 家长旅程

```
注册
└── 绑定孩子账号（孩子生成邀请码）

日常使用
├── 每晚推送：今日学习摘要（时长、任务完成、情绪状态）
├── 每周推送：战局周报 + 本周家长行动指南
└── 异常推送：孩子2天未登录 / 情绪预警

关键节点
├── 模考后：看到孩子预估分变化趋势
└── 志愿填报前：基于历次模考生成录取概率分析
```

### M2：试卷录入系统

#### M2.1 试卷上传（主流程）

**支持格式**：JPG/PNG/PDF，单次最多20页

**处理流程**：
```
上传图片
  → 图像预处理（去噪、矫正、增强对比度）
  → OCR识别（题目结构化：题号、题干、选项、学生答案、标准答案）
  → AI批改（逐题对比，标注正误）
  → Profiler Agent 深度分析（错误类型归因、认知断裂点定位）
  → 更新学生画像
  → 生成本次考试报告
```

**错误类型分类**（Profiler Agent输出）：
- **概念性错误**：根本没理解该知识点
- **迁移性错误**：理解了但不会在新场景中应用
- **粗心性错误**：会做但计算/书写失误
- **思维断裂**：推导到某步骤停滞，后续逻辑缺失

**处理时间要求**：
- 单张图片：≤ 60秒
- 完整试卷（20页）：≤ 3分钟
- 超时提示：进度条 + "处理中，可先看其他内容"

#### M2.2 单题快速录入（快捷流程）

**使用场景**：学生做题过程中遇到不会的题，当下立即录入

**交互设计**：
1. 首页悬浮按钮"拍一道题"
2. 拍照 → 识别题目 → 选择科目 → 确认
3. 立即进入苏格拉底对话（不等攒够一张卷子）
4. 对话结束后自动归入对应科目错题库

#### M2.3 手动录入成绩

**用途**：录入历次模考总分及各科分数，用于趋势分析和战局评估

**字段**：
- 考试名称（如：2025年3月第一次全市模考）
- 考试日期
- 各科分数
- 总分
- 年级排名（可选）

### M3：战局评估引擎

> 这是产品留存的核心钩子。用户第一次看到战局报告，必须感受到"这个系统真的懂我"。

#### M3.1 当前预估分计算

**计算逻辑**：
```
预估分 = Σ(各科预估分)

各科预估分 = 各科满分 × 综合得分率

综合得分率 = 
  历次模考得分率（权重40%）× 时间衰减系数
  + 知识点掌握度加权平均（权重40%）
  + 近期错题难度系数修正（权重20%）
```

**显示形式**：
- 预估分：`538分`（区间显示：`525～550分`）
- 各科分解：语文`105` / 数学`112` / 英语`118` / 综合`203`

#### M3.2 战局判断

| 状态 | 判断标准 | 展示颜色 | 展示文案示例 |
|------|---------|---------|------------|
| 绿色 | 预估分 ≥ 目标分 | 绿色 | "按当前进度，你大概率能达到目标，继续保持" |
| 黄色 | 目标分 - 预估分 ≤ 30分 | 黄色 | "还差28分，调整策略可以追上，重点看数学" |
| 红色 | 目标分 - 预估分 > 30分 | 红色 | "差距较大，必须做取舍——不是所有弱点都来得及补" |

#### M3.3 时间分析

```
剩余天数：63天
可用学习时长（估算）：63天 × 4小时/天 = 252小时
已识别需补知识点总时长：约340小时
结论：⚠️ 时间不够，必须取舍
```

### M4：作战决策系统

> 这是产品最核心的差异化功能。告诉学生"该打哪场仗、放弃哪场仗"。

#### M4.1 知识点ROI计算

```
ROI = (预期得分提升 × 高考权重系数) ÷ 补完所需天数

预期得分提升 = (1 - 当前掌握度) × 该知识点平均分值 × 0.8（保守系数）
高考权重系数 = 该知识点在高考中的历年平均分值占比
补完所需天数 = 基于当前掌握度 + 错误类型估算
```

#### M4.2 知识点三色分级

| 等级 | 判定标准 | 操作建议 |
|------|---------|---------|
| 🟢 必打 | ROI > 6 且剩余时间充足 | 本周优先投入，深度学习 |
| 🟡 选打 | ROI 3～6 或时间紧张 | 根据当周进度决定是否投入 |
| 🔴 放弃 | ROI < 3 或补完时间 > 剩余天数50% | 果断放弃，不要有心理负担 |

**放弃建议展示方式**（需谨慎措辞，避免打击信心）：
> "数列压轴题（第21题类型）：预计需要20天才能稳定掌握，而高考只有1道，分值6分。建议本阶段暂时跳过，把这20天用在更高回报的地方。这不是放弃，是聪明的选择。"

#### M4.3 每周作战清单

**格式**：每周一自动生成，内容精简，不超过5条

```
📋 本周作战清单（第8周，距高考63天）

🎯 本周必做（预计每项2小时）
  1. 解析几何：椭圆标准方程 + 焦点三角形（3道专项练习）
  2. 英语：完形填空逻辑词汇（每日10题，共50题）

⚡ 如果时间允许
  3. 物理：电磁感应计算（重点做选择题，放弃大题）

🚫 本周不做
  数列压轴题、化学有机推断（本阶段性价比过低）

💡 本周家长建议
  数学不需要催，英语每日10题需要监督打卡。
```

#### M4.4 长期作战地图

可视化展示未来每周的重点攻坚方向，让学生看到"完整的仗该怎么打"。

### M5：苏格拉底对话引擎

#### M5.1 对话模式切换

| 模式 | 触发条件 | 特征 |
|------|---------|------|
| **深度理解模式** | 距高考 > 90天 | 严格苏格拉底式，不给答案，多轮追问，引导学生自主推导 |
| **混合模式** | 30～90天 | 追问为主，遇到学生卡壳超过3轮时给出部分提示 |
| **冲刺模式** | < 30天 | 快速确认知识点理解 → 直接给出解题模板 → 强化记忆锚点 |

#### M5.2 Tutor Agent 核心规则

**铁律（任何模式下不违反）**：
1. 不直接给出最终答案或完整解题步骤
2. 当学生说"我不会"时，从最基础的概念开始追问
3. 只有当学生自己说出正确思路时，才给予肯定和推进
4. 每次追问只问一个问题，不连续发问

**追问策略（深度理解模式）**：
```
Step 1：还原题目 — "这道题在问什么？用自己的话说说"
Step 2：拆解条件 — "题目给了你哪些已知信息？"
Step 3：找突破口 — "你觉得第一步可以做什么？"
Step 4：暴露断裂 — 学生推导错误时 → "你这一步的依据是什么？"
Step 5：自我纠正 — 引导学生发现自己的逻辑错误
Step 6：总结归纳 — "你刚才用了什么方法解决这道题？"
```

#### M5.3 情绪感知层

**情绪识别触发词**：
- 焦虑类："学不完了"、"好慌"、"来不及了"、"没用"
- 崩溃类："不想学了"、"想放弃"、"我太差了"
- 愤怒类："这什么破题"、"学了有什么用"

**触发后处理逻辑**：
1. 暂停学术追问
2. 切换到情绪支持回应（非学术语气）
3. 若情绪词汇出现3次以上 → 触发家长端预警推送
4. 提供"今天就到这里吧"的退出选项，不强制继续

**情绪支持示例回应**：
> "听起来你今天压力很大。这很正常，高考前每个人都会有这种感觉。不用现在强迫自己继续——你今天已经做了不少了。要不要先休息一下？"

#### M5.4 对话结束后处理

对话结束时，Profiler Agent自动运行：
1. 分析本次对话中暴露的认知断裂点
2. 更新学生画像中的弱点日志
3. 修正该知识点的掌握度评分
4. 重新计算ROI，必要时更新作战清单

### M6：学生画像系统

#### M6.1 知识点体系

基于教育部高考考纲，覆盖6科：

- **数学**：函数、数列、向量、解析几何、概率统计、导数、空间几何（共42个二级知识点）
- **语文**：现代文阅读、古诗文、语言文字运用、写作（共18个）
- **英语**：阅读理解、完形填空、语法填空、写作（共12个）
- **物理/化学/生物**：各按模块细分

每个知识点记录：
- 掌握度（0.0～1.0）
- 错误类型分布
- 历史弱点日志
- ROI评分
- 分级状态（必打/选打/放弃）

#### M6.2 掌握度更新机制

```
新掌握度 = 旧掌握度 × 0.7 + 本次表现评分 × 0.3

本次表现评分：
  苏格拉底对话中自主推导成功 → 0.85
  苏格拉底对话中需要部分提示 → 0.55
  对话中未能自主推导 → 0.20
  试卷中答对（简单题） → 0.70
  试卷中答对（难题） → 0.90
  试卷中答错 → 0.15
```

### M7：家长端

#### M7.1 核心设计原则

- **不看题目细节**：家长不需要看学生的错题，那会引发干涉
- **看战局，看状态，看行动指南**：三件事
- **数字要少，文字要多**：家长需要判断，不需要数据堆砌

情绪状态：😊 稳定

## 9. 非功能性需求

- 图片上传失败：自动重试3次，支持断点续传
- AI服务超时：降级展示上次结果，异步重新计算

| AI对话响应延迟影响体验 | 中 | 高 | 流式输出（Streaming）+ 输入时显示"正在思考" |
| 学生不愿意被追问，流失 | 中 | 高 | 提供"我想直接看答案"的逃生出口（记录使用次数，不惩罚） |
| 家长付费意愿不足 | 中 | 高 | 免费提供战局报告（家长分享传播），付费解锁行动指南 |
| 与大厂产品竞争 | 高 | 中 | 差异化在"作战决策+苏格拉底+家长端"组合，大厂短期难复制 |
| AI生成内容准确性 | 中 | 高 | 知识点体系基于官方考纲，关键计算（预估分、ROI）有人工验证机制 |
| 情绪支持触发心理危机 | 低 | 极高 | 明确产品边界：不是心理咨询，危机词汇触发时引导联系专业人士或家长 |

### B. 核心术语定义

| 术语 | 定义 |
|------|------|
| 战局评估 | 基于当前学习数据对高考结果的综合预判 |
| 预估分 | AI基于错题分布和历次模考计算的当前能力对应分数 |
| ROI | 知识点学习回报率，衡量投入时间与预期得分提升的比率 |
| 三色分级 | 必打（绿）/选打（黄）/放弃（红）的知识点优先级判定 |
| 苏格拉底模式 | 通过追问引导学生自主推导答案的AI对话方式 |
| 弱点画像 | 学生在各知识点上的认知状态结构化记录 |
| 作战清单 | 每周精简的重点任务列表，包含必做、选做和明确放弃项 |
| 认知断裂点 | 学生推导过程中思维停滞或出现逻辑跳跃的具体位置 |
