## v1.2 重大变更说明

本版本是一次**根本性的战略重定位**，不是功能增量。

### 定位变更

| 维度 | v1.1（旧） | v1.2（新） |
|------|-----------|-----------|
| 产品定位 | 高考作战指挥系统 | 自我助学系统 / 个人学习成长档案 |
| 目标用户 | 高三学生及家长 | 全年级学生（初中+高中）及家长 |
| 核心隐喻 | 作战参谋（替你决策） | 一面镜子（帮你认识自己） |
| 时间维度 | 高考倒计时（1年） | 长期成长追踪（3-6年） |
| 核心资产 | 当周作战清单 | **永久学习档案 + 纵向认知轨迹** |

### 这次重定位解决了v1.1的三个致命问题

1. **政策合规风险** → 从"学科辅导"转为"自主学习工具"，站到政策鼓励的自主学习、反机械刷题一侧，大幅降低被认定为变相学科培训的风险。
2. **LTV过低** → 用户生命周期从1年延长至3-6年，单位经济模型从"一次性买卖"变为"长期陪伴"。
3. **预估分不可信** → 不再承诺高考绝对分数，预估分降级为"相对成长趋势"，回避被结果证伪的陷阱。

### 这次重定位带来的三个新挑战（本版重点解决）

1. **失去高三紧迫感后的冷启动钩子** → 见 第3章、M0。
2. **永久保存未成年人数据的合规高压线** → 见 第10章（数据合规专章，全新）。
3. **"数据多 ≠ 洞察多"** → 见 M6 纵向认知分析（全新核心模块）。

### 继承自v1.1的模块（保留并适配）

苏格拉底对话引擎、试卷录入、学生画像、家长端、遗忘曲线、执行力支持系统、今日目标——全部保留，但去除"高考倒计时""ROI放弃决策"等高三专属逻辑，改造为全年级适用。

## 1. 产品概述

### 1.1 产品定位

**自我助学系统**是一个面向全年级学生（初中、高中）的**个人学习成长档案与自主学习工具**。

它把学生从初中到高中的每一张试卷、每一道错题、每一次认知突破，永久保存为一份不断生长的"学习成长档案"，并通过AI帮助学生认识自己的学习模式、修复认知断点、持续自主进步。

核心价值主张：

> 它不替你决定该学什么。它帮你看清——你是怎么学的，你在怎样变好。

### 1.2 产品哲学

- **不是辅导班，是一面镜子**：我们不替代老师教学，而是帮学生照见自己的学习轨迹。
- **不承诺分数，记录成长**：我们不预测你能考多少分，而是忠实记录并呈现你的进步。
- **数据属于学生自己**：每一份学习档案是学生的个人资产，可导出、可带走、终身可追溯。

### 1.3 现有工具的根本缺陷

- **数据是碎片的，不是连续的**：刷题软件记录单次结果，没人保存学生三年的认知演变。
- **只诊断当下，不追踪历史**：没人能回答"我这个知识点三年来到底有没有进步"。
- **只给答案，不修复认知**：错题对完答案就完了，下次还错。
- **数据属于平台，不属于学生**：换个产品，学习历史归零。

### 1.4 三大支柱

```
支柱一：永久学习档案
  每张试卷、每道错题永久保存，构成学生专属的成长史

支柱二：纵向认知分析
  跨年度追踪同一知识点的掌握演变，识别个人学习模式

支柱三：苏格拉底自助引擎
  不给答案、追问引导，修复认知断点（继承自v1.1）
```

三者形成闭环：试卷沉淀为档案，档案揭示长期模式，苏格拉底修复当下断点，新的学习又沉淀为档案。

## 2. 用户研究

### 2.1 目标用户

**主要用户：全年级学生（初一至高三）**
- 年龄：12～18岁
- 分层需求：
  - 初中生：建立学习习惯、积累错题、认识自己的薄弱点
  - 高一高二：长期追踪、知识体系查漏补缺
  - 高三：冲刺期的重点突破（继承v1.1的部分能力，但不再是唯一焦点）

**次要用户：学生家长**
- 核心诉求：看到孩子长期、连续的成长，而不只是单次分数
- 二孩家庭：多孩子统一管理

### 2.2 全年级用户的痛点

| 痛点 | 具体表现 | 我们的解法 |
|------|---------|-----------|
| 错题攒了就丢 | 错题本写了不看，纸质易丢 | 永久数字档案，随时可查 |
| 不知道自己在进步吗 | 努力了但看不到变化 | 纵向成长曲线，进步可视化 |
| 同样的错反复犯 | 这次错的，下次还错 | 遗忘曲线 + 认知模式识别 |
| 学了不理解 | 背了公式不会用 | 苏格拉底引导自主推导 |
| 不知道自己是怎样的学习者 | 没有自我认知 | 个人学习模式画像 |
| 升学换工具历史归零 | 初中数据进高中就没了 | 跨学段永久保存 |

### 2.3 关键场景

**场景一：第一次使用的"顿悟"体验（冷启动钩子）**
> 初三学生小林上传一张数学卷。3分钟后，系统没有只告诉他"错了3道"，而是说："你这3道题都错在同一个地方——设了未知数之后，列方程时漏掉了一个条件。我们一起看看第一道。"接着进入苏格拉底追问，小林自己发现了那个漏掉的条件，"啊"地一下想通了。这个顿悟瞬间，让他第一次用就留下了。

**场景二：跨年度的成长回看**
> 高一的小林某天打开"我的成长档案"，看到一条曲线：初二时他的"二次函数"知识点掌握度是0.3，初三0.55，现在0.8。系统说："二次函数是你进步最大的知识点，从初二到现在，你在它上面犯的错从'概念不清'变成了偶尔的'计算粗心'——这是真正的理解了。"小林第一次直观看到自己两年的成长。

**场景三：个人学习模式识别**
> 系统在积累了小林半年数据后告诉他："发现一个规律：你在需要3步以上推理的题目里，正确率会明显下降，而单步题几乎全对。这说明你的薄弱不在知识点本身，而在多步骤的思路保持。建议练习时把长题拆成小步骤写下来。"这是任何单次分析给不出的洞察。

**场景四：家长看到的是成长，不是分数**
> 小林妈妈打开家长端，看到的不是"这次考了多少分"，而是"小林本月在物理上的薄弱点从7个减少到4个，已经连续学习23天"。她看到的是趋势和坚持，焦虑被缓解。

## 3. 产品目标与成功指标

### 3.1 冷启动钩子设计（重定位后的关键）

失去高三紧迫感后，留存依赖两层钩子：

**第一层 · 即时顿悟（不依赖长期数据，第一次就能给）**
- 上传一张试卷 → 不只列错题，而是指出"你这些错误背后的同一个断点"
- 一次苏格拉底"想通了"的体验
- 目标：第一次使用就产生"这个东西真的看懂了我"的感受

**第二层 · 长期增值（用得越久越离不开）**
- 学习档案随时间增长，纵向洞察越来越准
- 迁移成本随数据积累指数上升
- 目标：用满3个月后，用户不愿意丢掉自己的成长档案

### 3.2 成功指标（OKR）

#### Objective 1：第一次就打动用户

| 关键结果 | 目标值 |
|---------|-------|
| 首次使用完成一次完整苏格拉底对话比例 | ≥ 50% |
| 次日留存率 | ≥ 40% |
| 首周"顿悟时刻"触发率（自主推导成功） | ≥ 60% |

#### Objective 2：长期留存与档案积累

| 关键结果 | 目标值 |
|---------|-------|
| 3个月留存率 | ≥ 25% |
| 人均累计上传试卷数（3个月） | ≥ 15张 |
| 跨学期留存率（升年级不流失） | ≥ 50% |

#### Objective 3：AI效果可量化（修正v1.1虚荣指标）

| 关键结果 | 目标值 | 说明 |
|---------|-------|------|
| 错题二次正确率 | ≥ 65% | 替代"对话时长"这一有害指标 |
| 自主推导成功率 | ≥ 60% | 苏格拉底引导的真实效果 |
| 遗忘曲线回顾通过率 | ≥ 70% | |
| 纵向洞察被用户标记"有用"比例 | ≥ 50% | 验证"数据→洞察"转化 |

> **重要修正**：v1.1 的"对话时长≥8分钟"是有害指标——苏格拉底高效应让学生更快想通。v1.2 删除该指标，改用"自主推导成功率"和"二次正确率"衡量真实效果。

## 4. 核心差异化：永久学习档案

> 这是 v1.2 的灵魂，也是产品真正的护城河。

### 4.1 档案的构成

学生的永久学习档案包含：

```
个人学习成长档案
├── 试卷库（永久保存所有上传的试卷原图 + 结构化数据）
├── 错题库（每道错题 + 当时的认知分析 + 苏格拉底对话记录）
├── 知识点掌握度时间序列（每个知识点逐月演变曲线）
├── 错误类型演变（概念→迁移→粗心的转化轨迹）
├── 个人学习模式画像（识别出的独特规律）
└── 成长里程碑（突破时刻、连续学习记录）
```

### 4.2 为什么"永久"是壁垒

- **不可复制**：竞品无法获得用户过去几年的连续数据
- **迁移成本极高**：用户用得越久，离开的代价越大
- **越用越准**：纵向数据越长，个人模式识别越精确
- **数据资产归属学生**：可导出为PDF成长报告，强化"这是我的东西"

### 4.3 "数据→洞察"的转化（避免数据沉睡）

存储不等于价值。每一份积累的数据必须能产出洞察：

| 数据积累 | 产出的洞察 |
|---------|-----------|
| 单次试卷 | 本次认知断点 + 同类错误归因 |
| 1个月 | 高频薄弱知识点 + 错误类型分布 |
| 1学期 | 知识点掌握演变 + 个人学习模式初判 |
| 1学年+ | 跨年度成长曲线 + 稳定的学习者画像 |
| 跨学段 | 初中→高中的知识衔接断层识别 |

详细分析能力见 M6。

## 5. 用户旅程地图

### 5.1 学生旅程（全年级通用）

```
[发现] → [注册] → [首次顿悟] → [习惯养成] → [长期增值] → [升学延续]

发现
└── 朋友分享成长报告 / 老师推荐 / 搜索"错题本""自主学习"

注册
├── 填写：年级、地区、科目（按需，不强制目标院校）
└── 立即引导上传第一张试卷

首次顿悟（冷启动关键，5分钟内）
├── 上传一张试卷
├── 系统指出"这些错误背后的同一个断点"
└── 一次苏格拉底对话 → "想通了"的瞬间

习惯养成（第一层留存）
├── 每天今日目标（最重要的一件事）
├── 错题随手拍 + 即时苏格拉底
├── 遗忘曲线定期回顾
└── 试卷持续沉淀进档案

长期增值（第二层留存，3个月后显现）
├── 成长曲线可视化（看到自己在变好）
├── 个人学习模式画像逐渐清晰
└── 纵向洞察越来越准

升学延续（最强壁垒）
└── 升年级/升学段，档案无缝延续，历史不归零
```

### 5.2 家长旅程

```
注册 → 绑定孩子（支持多孩子）
日常 → 每晚微信一句话日报（看成长，不看分数）
      → 5类风险预警
关键 → 学期/学年成长报告（孩子长期进步的可视化）
```

### M1：用户体系（全年级）

#### M1.1 学生注册
- 手机号 + 验证码（未成年人需监护人手机号验证，见第10章）
- 年级（初一～高三）
- 地区（用于匹配教材版本/考纲）
- 关注科目
- **不强制**填写目标院校（高三用户可选填）

#### M1.2 家长注册与多孩子
- 绑定孩子邀请码，支持多个孩子切换
- 监护人身份用于满足未成年人数据合规要求（见第10章）

#### M1.3 升年级/升学段处理
- 学年更替时，档案自动延续，年级标签更新
- 初中升高中：保留全部历史，标注学段切换点，用于衔接分析

### M2：试卷录入 + 永久归档（🆕 永久归档）

#### M2.1 上传与归档流程
```
上传试卷图片/PDF
  → 图像预处理
  → OCR结构化识别
  → AI批改
  → 认知分析（Profiler Agent）
  → 【永久归档】原图 + 结构化数据 + 分析结果存入档案
  → 初始化遗忘曲线
  → 沉淀进知识点时间序列
```

#### M2.2 永久归档设计
- **原图永久保存**（冷热数据分层，见第7章控制成本）
- **结构化数据永久保存**（题目、答案、知识点、错误类型、时间戳）
- 每张试卷标注：年级、学期、考试类型、上传时间
- 支持按时间轴、科目、知识点多维度回溯

#### M2.3 单题快速录入
- 随手拍一道题 → 即时分析 → 进入苏格拉底
- 同样永久归档

### M3：单次试卷认知分析（改造，去预估分承诺）

> **重要改造**：删除 v1.1 的"高考预估分"绝对数字承诺。

#### M3.1 单次分析输出
- 本次错误清单 + 错误类型分类
- **核心：找出错误背后的"共同断点"**（这是冷启动顿悟钩子）
  > "你这3道题都错在同一个地方：列方程时漏掉隐含条件"
- 各错题的苏格拉底入口

#### M3.2 相对成长反馈（替代绝对预估分）
- 不说"你能考X分"
- 改说相对趋势：
  > "本次数学薄弱点比上次减少2个"
  > "二次函数掌握度从0.6提升到0.7"
- 高三用户可选开启"分数趋势"（基于模考相对变化，仍不承诺高考绝对分）

### M4：苏格拉底对话引擎（继承 v1.1）

完整继承 v1.1 的 M5：
- 三种模式（深度/混合/快速），按年级和场景切换（不再绑定高考倒计时）
- Tutor Agent 铁律：不直接给答案、每次一问、自主推导才推进
- 逃生出口（直接看答案，不惩罚）
- 情绪感知层（识别焦虑/崩溃，暂停学术，必要时家长预警）
- 语音输入（Phase 2）
- 30秒首次追问 → **修正为：先给通用占位追问（即时），精准追问异步补上**（解决 v1.1 单题场景SLA矛盾）

### M6：永久档案 + 纵向认知分析（🆕 核心新模块）

> 这是把"永久数据"转化为"永久洞察"的引擎，是 v1.2 真正的差异化。

#### M6.1 知识点掌握度时间序列
- 每个知识点记录逐月掌握度快照
- 生成纵向曲线：看到某知识点几个月/几年的演变

#### M6.2 错误类型演变追踪
- 追踪同一知识点的错误类型如何变化
- 例："二次函数：概念性错误(初二) → 迁移性错误(初三) → 粗心性错误(高一)"
- 错误类型从"概念"转向"粗心"= 真正理解了，这是有力的成长证据

#### M6.3 个人学习模式识别
积累足够数据后（建议 ≥ 1学期），识别学生独特的学习模式：

| 模式维度 | 识别示例 |
|---------|---------|
| 推理链长度 | "你在3步以上推理的题正确率骤降" |
| 题型偏好 | "你概念题强，应用题弱" |
| 错误时段 | "你下午做题的正确率明显低于上午" |
| 知识衔接 | "高中函数的弱，根源在初中的方程基础" |

实现方式：基于档案数据，由分析Agent定期（每月）运行，输出结构化模式洞察。

#### M6.4 跨学段衔接分析
- 初中升高中时，分析知识断层
- 例："你即将学的高中三角函数，依赖初中的相似三角形，而那是你初中的薄弱点，建议先补"

### 7.1 整体架构

```
├── 对象存储（试卷原图，冷热分层）
└── pgvector（题目向量，Phase 3）
```

### 7.2 永久存储的成本控制（新增关键设计）

永久保存试卷原图，存储成本随时间线性增长，必须分层：

| 数据层 | 内容 | 存储策略 |
|--------|------|---------|
| 热数据 | 近3个月试卷原图 + 全部结构化数据 | 标准OSS，快速访问 |
| 温数据 | 3个月-1年试卷原图 | 低频访问存储 |
| 冷数据 | 1年以上试卷原图 | 归档存储（访问时解冻） |
| 永久层 | **所有结构化数据**（题目/分析/掌握度） | 永久保留在数据库，体积小、价值高 |

**核心原则**：结构化数据（小而有价值）永久在线；原图（大）分层降冷。即使原图归档，纵向分析依然可用，因为分析依赖结构化数据而非原图。

### 7.3 技术选型

继承 v1.1（FastAPI + PostgreSQL + Redis + 阿里云OSS + Claude Vision/API），新增：
- 时间序列数据：PostgreSQL 时间序列表（掌握度快照），数据量大后可评估 TimescaleDB
- 对象存储分层：阿里云OSS 标准/低频/归档三档

## 9. AI引擎设计

继承 v1.1 的 Profiler Agent、Tutor Agent，新增两个：

### 9.1 共同断点分析 Prompt（冷启动钩子核心）
```
你是一位资深教师，分析这名学生本次试卷的所有错题。

【任务】不要逐题孤立分析。找出这些错误背后是否存在
【共同的认知断点】——即多道题错在同一个根本原因上。

【错题列表】
{wrong_questions}

【输出】
1. 是否存在共同断点（是/否）
2. 若有，用一句话点出（如"列方程时反复漏掉隐含条件"）
3. 选最典型的一道题作为苏格拉底切入点
4. 若无共同断点，列出最值得优先攻克的1个知识点

语气：像一个真正看懂了学生的老师，让学生产生"原来如此"的感受。
```

### 9.2 纵向模式识别 Prompt（M6.3）
```
你是学习行为分析专家。基于这名学生【至少一学期】的学习档案，
识别其独特的学习模式。

【档案数据】
{historical_data}  // 含各知识点掌握度时间序列、错误类型演变、答题时段等

【输出JSON】
{
  "patterns": [
    {
      "type": "reasoning_chain|topic_preference|time_of_day|cross_stage",
      "description": "用学生能懂的话描述这个规律",
      "confidence": 0.0~1.0,
      "evidence": "支撑这个判断的具体数据",
      "suggestion": "针对这个模式的具体建议"
    }
  ]
}

只输出置信度>0.6的模式。数据不足时宁可少输出，不要编造规律。
```

## 10. 数据合规与未成年人保护（全新专章）

> 永久保存大量未成年人学习数据，是本产品最敏感的合规高压线。本章为强制要求，非建议。

### 10.1 法律依据
- 《中华人民共和国个人信息保护法》：不满14周岁未成年人个人信息属于敏感个人信息
- 《未成年人保护法》网络保护专章
- 《未成年人网络保护条例》

### 10.2 强制合规要求

| 要求 | 实现 |
|------|------|
| 监护人同意 | 不满14岁用户，注册需监护人手机验证 + 单独同意条款，记录存证（guardian_consents表） |
| 单独的处理规则 | 制定并公示《儿童个人信息处理规则》 |
| 最小必要原则 | 只收集学习相关数据，不收集无关信息 |
| 数据加密 | 试卷图片、个人信息加密存储与传输 |
| 永久保存的额外义务 | 明确告知保存期限/目的；提供**随时删除/导出**入口 |
| 数据可携带权 | 学生/监护人可一键导出全部档案 |
| 删除权 | 用户要求删除时，彻底删除（含归档层） |
| 数据本地化 | 全部存储于境内节点 |

### 10.3 "永久保存"的合规边界
- "永久"是**为用户保存**，不是平台无限期占有
- 用户随时可删除、可导出、可注销
- 注销账户时，按法律要求彻底清除（保留必要的法定记录除外）
- 不得将未成年人数据用于个性化广告

### 10.4 安全责任
- 数据泄露是公司级风险，需：访问最小权限、审计日志、定期渗透测试、数据脱敏用于内部分析
