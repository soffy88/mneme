# Mneme 产品价值重构

> **性质**：对 `MNEME_MASTER_DESIGN.md` 的**产品表达层修正**。
> **不动**：技术内核（KT/FSRS/forgetting-aware/七大机制/3O 分层/数据模型/API 契约）全部保留有效。
> **修正**：重排产品定位与卖点表达，补齐两个家长刚需却缺失的模块。
> **依据**：2026 艾瑞 GenAI+教育报告、学习机市场实证、知乎家长真实评价（见 §1 引用）。
> **日期** 2026-06 ｜ **状态** Master 的产品层修正案，经确认后并入 Master 第一部分与路线图。

---

## 1. 为什么要重构：实证暴露的三个错位

我们一路从"试卷分析"滚雪球到 Mneme，技术越做越精，却从未验证学生家长到底买不买单。实证一查，三个错位刺眼：

**错位 1 · 家长付费的两大刚需，我们最弱。**
艾瑞 2026 调研显示家长最看重的教育 GenAI 功能是**作业批改（42.6%）和 AI 题库（42.5%）**，位居前三。学习机店里家长最关注题型题量、试卷权威性、扫一扫判作业、薄弱知识点"卫星图"。而 Mneme：刻意回避题库（怕版权），批改只是入口非主打。**我们把家长第一位的需求当成了配角。**

**错位 2 · "举一反三"是真痛点，行业普遍做不到——这是机会。**
避坑指南直接点破：80% 的学习机 AI 只能做"错题本"，无法真正举一反三，只是把错题归类、没有溯源知识点；真正该做的是"函数学不好，AI 能自动关联方程、几何"。**这正是我们 KC 前置链 + 知识追踪能做、别人没做好的事**——但我们没把它包装成家长听得懂的卖点。

**错位 3 · 学生持续用的是"爽点"，Mneme 全是"苦点"。**
店里观察学生喜欢和立体 AI 人像交流、课程动画游戏互动、扫课本详解。而 Mneme 给学生的全是反人性的苦：苏格拉底追问（累）、交错练习（更难）、检索练习（被迫回忆）、努力错觉对抗（讲道理）。**这些机制学习科学上都对，但一个没有任何爽点的产品，学生不会打开第二次。**

**最致命的警告：连大厂都栽在"批改不准"。**
学而思被家长吐槽 AI 批改识别不出错误、把对的判成错的、数学题给英语答案、给超纲答案。这验证了我们"确定性内核兜住算"的方向对，也警告：**批改/识别这关做不到位，产品直接出局。**

### 但实证也证明我们三件事做对了

- **逐字思路回顾 = 家长真想要**：科大讯飞因"App 能看到 AI 引导过程的逐字回顾，家长不在身边也能看孩子思路怎么演变"被称赞——正是我们 `decision_trail` + 苏格拉底记录能做的。
- **本省真题适配是硬门槛**：新课标下必须适配本省中高考，否则方向全错——我们"单省单科做深（广东数学）"方向对。
- **错题 + 考点整理是功能偏好前二**：我们的档案 + KC 体系是地基。

## 2. 重构的核心判断

> Mneme 不能是"一个有很多学习科学机制的工具"，而要是"**一个让家长看到效果、让学生愿意打开的提分助手**"。
> **机制是引擎盖下面的东西，不是卖点。我们之前一直在卖引擎，没卖车。**

由此把产品重排为**三层结构**：

```
┌─────────────────────────────────────────────┐
│ 卖点层（对外说什么 · 对应付费理由）           │
│   批得准 · 举一反三 · 提分看得见 · 讲解爽     │
├─────────────────────────────────────────────┤
│ 必补层（家长刚需但我们缺）                    │
│   活的变式题（绕版权）· 学生爽点系统          │
├─────────────────────────────────────────────┤
│ 引擎层（藏在下面 · 让卖点"真的有效"）         │
│   KT/FSRS · 确定性内核 · 交错/检索/识别 · 档案│
└─────────────────────────────────────────────┘
```

---

## 3. 卖点层：对外说什么

**原则**：卖点必须是家长学生一句话听懂、且对应明确付费理由的东西。引擎术语（BKT、FSRS、知识追踪）永远不出现在卖点里。

| 给谁 | 卖点（对外话术） | 实证依据 | 底层靠什么引擎 |
|------|----------------|---------|--------------|
| 家长 | **拍照批改，而且批得准** | 批改第一需求；大厂栽在不准 | 确定性内核兜数值 + 同源自检 + 手动纠错 |
| 家长 | **真正举一反三，不只是错题归类** | 80% 产品做不到，明确痛点 | KC 前置链 + 知识追踪溯源到根知识点 |
| 家长 | **提分轨迹看得见 + 思路逐字回顾** | 讯飞因此被赞 | 成长曲线（long_term_mastery）+ 苏格拉底 trail |
| 家长 | **广东中高考考点，刷得准** | 本省适配是硬门槛 | 单省单科 KC 字典 + 权重 |
| 学生 | **不会的题，30 秒讲明白还能看动图** | 学生爱"扫课本详解"+动画 | edulab 式可视化讲解（M-D）|
| 学生 | **按你的弱点出题，不刷废题** | 题库需求 + 不刷无效题 | 变式题引擎（§5）+ 掌握度 |

**家长端首屏话术改写**（从"数据"到"效果"）：
> 旧：当前预估 538 分 / 知识点掌握度雷达图（冷冰冰数据）
> 新：「本月孩子的数学薄弱点从 7 个减到 4 个，函数这块已经追上来了。点开看 AI 是怎么一步步引导他想通的。」（效果 + 思路回顾）

---

## 4. 引擎层：藏好，别当卖点

KT/FSRS、确定性内核、交错/检索/识别维度、努力错觉、永久档案——**全部继续做，但对用户隐形**。

- 它们让"举一反三""提分轨迹""按弱点出题"真的有效，是卖点成立的底层支撑。
- 你永远不对家长说"我们用了贝叶斯知识追踪"；只说"能溯源到根知识点、举一反三"。
- 学生感觉不到"这是交错练习"，只觉得"今天的练习刚好有点挑战又不至于做不出"。
- 努力错觉对抗不说教，藏成一句正反馈："这道难题你啃下来了，记得最牢的往往就是这种。"

**这是本次重构对引擎层的唯一要求：不改技术，改表达——把术语翻译成效果。**

## 5. 必补模块一：活的变式题引擎（绕过版权）

### 5.1 为什么必补
家长第二大需求是 AI 题库（42.5%），我们一直回避（怕真题版权）。但"不刷题"的教育产品在中国卖不动。

### 5.2 解法：不存真题，活生成变式题
基于学生薄弱 KC，用 LLM 生成"同考点变式题"，确定性内核保证答案正确。**别人的题库是死的、按章节排的；我们的题是活的、按你此刻的薄弱点生成的，且答案由内核保证对。** 这既满足刷题需求、规避版权，又正好复用我们的求解内核——是我们独有的优势。

### 5.3 3O 归层（复用已有资产）
- `oprim.generate_variant`（单 LLM 调用：给定 KC + 难度，出题型骨架与参数）
- `oprim.solve_*`（内核求解，**保证答案与解析正确**，已有）
- `oprim.evaluate_diagram` / `kernel_to_plot2d`（配图，已有）
- `oskill.generate_practice_set`（组合：选薄弱 KC → 生成变式 → 内核验答 → 配可视化 → 难度校准）
- `omodul.practice_workflow`（业务事务：一次出一组个性化练习，支柱 `{fingerprint, report}`；fingerprint 去重避免重复出同题）

### 5.4 关键质量门
- **内核可解则答案必由内核给**（确定性优先红线，已有）；内核不可解的题型不生成变式，避免出错题。
- 变式题难度与学生当前掌握度匹配（太难打击、太易无效）。
- 题目与广东中高考题型一致（KC 字典的 question_types 约束）。

### 5.5 对家长的呈现
> 「针对孩子的薄弱点'椭圆'，今天生成了 3 道同类型练习，答案和解析都经过精确验证，不会出错。」

## 6. 必补模块二：学生爽点系统

苦的机制必须包一层糖，否则学生不打开第二次。三个爽点，原则是"高级的游戏化，不幼稚"。

### 6.1 可视化讲解（最大爽点，已在 M-D，提升为学生侧核心）
edulab 式"丝滑图示"——抽象变直观，"看见"本身就是最强的爽点。函数图象、解析几何动图、立体几何可旋转 3D。**这是学生愿意主动打开 Mneme 的第一理由**。优先级从 Phase 2 提到与 MVP 并重。

### 6.2 即时正反馈 + 社会比较
- 答对的即时动效（克制，不幼稚）。
- 社会比较文案（学生在意排名）：「这个考点你已经超过 80% 的同省同学」——数据来自掌握度分布，真实不浮夸。
- 攻克薄弱点的里程碑提示：「函数这块，你从 3 个月前的薄弱追到了班级中上。」

### 6.3 考点地图（KC 字典可视化成"闯关地图"）
把广东数学 KC 字典做成可视化知识地图：每个 KC 是一个节点，按掌握度点亮（灰→黄→绿），前置链是连线。学生看到自己"点亮"了多少考点、还有哪些黑着——**把抽象的掌握度变成可见的领地扩张**。这是 Streak 之外更强的长期动力，且天然复用我们的 KC 字典 + 前置链 + 掌握度，几乎零额外数据成本。

### 6.4 爽点与苦点的配比原则
- 每次"苦"（苏格拉底/检索/难题）后必跟一个"爽"（讲明白了/点亮考点/超过 X%）。
- 努力错觉对抗藏进正反馈，不单独说教。
- 新手期降低苦度（少交错、多即时正反馈），建立习惯后再加挑战。

## 7. 3O 归层增量（本次重构新增元素）

| 元素 | 层 | 机制 |
|------|----|------|
| `generate_variant` | oprim（单 LLM 调用） | 变式题 |
| `generate_practice_set` | oskill | 变式题 |
| `practice_workflow` | omodul `{fingerprint,report}` | 变式题 |
| `compute_peer_percentile` | oprim（纯计算：掌握度分布百分位） | 社会比较 |
| 考点地图渲染 | 前端（不入主库，复用 KC 字典数据） | 考点地图 |

数据模型增量：
```sql
CREATE TABLE practice_sets (        -- 变式题练习组
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  student_id UUID, target_kc VARCHAR(100),
  items JSONB,                      -- [{question, answer, solution, plot_data}]
  fingerprint VARCHAR(64),
  kernel_verified BOOLEAN,          -- 答案是否经内核验证
  created_at TIMESTAMPTZ DEFAULT now() );
-- kc_mastery 已有数据足够算 peer_percentile，无需新表
```

## 8. 功能优先级重排（修正 Master §12 路线图）

**旧 MVP 核心闭环**（工程视角）：上传卷 → 内核 → 薄弱排序。
**新 MVP 核心闭环**（用户价值视角）：

```
拍照批改(准) → 错因溯源(举一反三) → 可视化讲解(爽点) 
→ 按薄弱点变式练习(题库需求) → 成长看得见(家长付费)
```

**重排后的 MVP 必含**（对用户可见的价值）：
1. 拍照批改 + 准确性兜底（Epic 3 + Epic 10 确定性内核**提前**）
2. 错因溯源到根知识点（Epic 4 + KC 前置链，**包装成"举一反三"对外**）
3. 可视化讲解（Epic 11 的 M-D **提前到 MVP**，这是学生爽点）
4. 变式题练习（**新 Epic 13**，复用内核）
5. 家长成长摘要 + 思路回顾（Epic 6 的 overview + 苏格拉底 trail **提前**）

**保持隐形、继续做但不催**：交错/检索/识别维度/努力错觉（Epic 12，引擎层）、纵向分析、DKT。

**新增 Epic 13 · 变式题与爽点系统**：
```
13.1 [P0] oprim.generate_variant + 内核验答（绕版权刷题）
13.2 [P0] oskill.generate_practice_set + omodul.practice_workflow
13.3 [P1] oprim.compute_peer_percentile + 社会比较文案
13.4 [P1] 前端考点地图（KC 字典可视化闯关）
13.5 [P1] 即时正反馈 + 爽苦配比策略
DoD：变式题答案 100% 经内核验证；考点地图正确反映掌握度
```

**对 Master §12 的调整**：把 Epic 10（确定性内核）、Epic 11 的 M-D（可视化讲解）、Epic 6 的家长摘要从 Phase 2 **提前进 MVP**；新增 Epic 13。原因：它们是对用户可见的卖点，不是引擎。

## 9. 修正后的一句话定位

> **Mneme（学鉴）：拍照批改批得准、按孩子的弱点出题讲题、提分轨迹看得见的 AI 提分助手。**
> 对家长：准的批改 + 举一反三 + 看得见的进步。
> 对学生：不会的题 30 秒看懂动图，按你的弱点练，不刷废题。
> （引擎盖下：知识追踪 + 间隔重复 + 永久档案 + 学习科学机制，让以上每一句都真的成立。）

## 10. 这次重构改了什么、没改什么

**改了（产品表达层）**：
- 定位从"自我助学工具"→"AI 提分助手"（对外）。
- 卖点从"机制"→"效果"（批准/举一反三/提分可见/讲解爽）。
- 补两个模块：活的变式题、学生爽点系统。
- 路线图：确定性内核、可视化讲解、家长摘要提前进 MVP；新增 Epic 13。

**没改（技术内核，Master 第二三部分全部有效）**：
- KT/FSRS/forgetting-aware 算法内核。
- 七大机制（只是把它们藏到引擎层，不当卖点）。
- 3O 分层、数据模型、API 契约、合规专章。
- 永久档案的护城河定位。

**本质**：我们造的引擎是好的，这次是给引擎装上车壳、方向盘和座椅——让它变成学生家长真的想买、想开的车。
