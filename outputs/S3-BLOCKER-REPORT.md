# W2a S3 阻断报告 —— 真人 pilot 缺前端入口

**结论**：S3（真人 pilot）**不可执行**。W8–W12 结构性未达（红），**非失败** —— 缺的是真人作答的前端入口，不是引擎/管道能力。

## W8–W12 要证什么
一个**真实儿童完整用过一遍**：真人在界面上对引擎出的题**实际作答**（会答错、会犹豫、非满分），产生 P1–P5 探针。关键：**答案来自真人，expected 对答题方不可见**（这正是 W3 守的东西）。

## 真人作答需要什么
1. **前端 UI / 最小可用作答界面**：引擎（qwen 驱动）经 `/mcp` 出题 → 真人在界面看到题、输入答案 → 答案经 `/mcp/SubmitAnswer` 回引擎。
2. **人在环异步**：引擎 PoseQuestion 后**暂停等真人**，真人答完引擎再续跑（非自动闭环）。
3. **P4/P5 需跨时**（FSRS 复习到期）或多次会话。

## 现状缺什么
- W2a 全程走 **HTTP / tool 直调，无前端 UI**。真前端在 `mneme-web`（独立仓），**未与 tutor-loop 接线**。
- tutor-loop 现为**自动闭环**（scripted / qwen 自出自答），**无真人输入口**。
- **无人在环异步机制**（引擎不等真人，自己往下跑）。

## 已证 vs 未证
- **已证（operator / LLM 能力）**：qwen 能驱动引擎循环、选工具、出题，经真 HTTP 写库；引擎装配 / 工具面 / guard / 合规删除全绿。
- **未证（真人链路）**：真实儿童作答产生 P1–P5。需要前端 + 人在环 + 真人 —— CC 造不出，也不该再造代答脚本。

## 反例代码已删
`s3_pilot.py`（把 expected 种进 qwen prompt 让其自答）**已删除** —— 那是把答案喂给答题方的反例，留树里是隐患（同 A3 死代码逻辑）。`qwen adapter`（`351d8cf`）**保留**：引擎驱动 / 出题正确，错只在代答。

## 解锁路径（W2b+）
接 `mneme-web` 前端到 tutor-loop（人在环：PoseQuestion → 前端展示 → 真人输入 → SubmitAnswer），真人内测跑 P1–P5，W8–W12 方可转绿。
